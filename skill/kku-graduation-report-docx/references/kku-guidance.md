# KKU-specific guidance

Use this mapping when the user's professor sample is the King Khalid University Applied College report format.

## Priorities
1. Match the provided sample structure exactly.
2. Improve the writing quality, formatting, and consistency.
3. Keep all claims grounded in the codebase and uploaded sample files.

## Known chapter expectations
- Chapter 1 introduces the system and basic project framing.
- Chapter 2 compares similar apps/websites and related work.
- Chapter 3 covers users, requirements, and use case specification.
- Chapter 4 focuses on UML and dynamic/static design.
- Chapter 5 presents data modeling and interfaces.
- Chapter 6 explains the database design in more detail.
- Chapter 7 presents user interface screens.
- Appendix contains selective important code snippets, not a full source dump.

## Common quality upgrades to apply
- Fix broken or inconsistent table of contents formatting.
- Avoid repeating nearly identical explanatory paragraphs under every module.
- Keep headings and capitalization consistent.
- Add brief but meaningful explanations after each diagram and figure.
- Use one terminology set for actors, entities, pages, and modules.
- Distinguish clearly between:
  - analysis
  - design
  - database
  - interface presentation
  - code appendix
